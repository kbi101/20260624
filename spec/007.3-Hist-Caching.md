# Phase 007.3 — Web Query Cache for HIST

## Overview

HIST's ingestion pipeline has three expensive steps: page fetch (Playwright, ~5s), entity extraction (Ollama LLM, ~60-120s), relation extraction (Ollama LLM, ~90-180s). For pages already ingested or recently seen, re-running any of these is wasteful. This phase adds a three-tier cache to eliminate redundant work.

## Design Principles

1. **Aggressive caching** — Wikipedia content is quasi-static; a 24-hour TTL on page text is safe and saves significant time.
2. **Persistence across sessions** — all caches survive process restart via disk-backed JSON files.
3. **Transparent** — callers use the same `fetch_full_page()`, `extract_facts()` APIs; cache hits are invisible.
4. **Minimal memory footprint** — caches load on demand, not eagerly at startup.

---

## Three-Tier Cache Architecture

### Tier 1 — Page Text Cache

| Property | Value |
|---|---|
| Storage | Disk: `data/hist_cache/page_cache/` (one JSON per URL slug) |
| TTL | **24 hours** from `fetched_at` timestamp |
| Key | Last URL path segment normalized (e.g. `Robert_E._Lee.robert_e_lee.json`) |
| Content | Fetched wiki text (~6000 chars readability-extracted, from Morphos WebFetch) |

```jsonc
{
  "url": "https://en.wikipedia.org/wiki/Robert_E._Lee",
  "fetched_at": 1750001234.5,
  "text": "[[no-title]]\nConfederate States general..." // ~6000 chars
}
```

**Hit:** Return cached `text` immediately — saves Playwright launch + network (~5s) and bypasses the 3s throttle.

**Miss / stale (>24h):** Call `WebFetch.execute(url)`, persist fresh text, update timestamp.

### Tier 2 — Extraction Cache

| Property | Value |
|---|---|
| Storage | Disk: `data/hist_cache/extraction_cache/` (one JSON per URL slug) |
| TTL | **No expiration** — Wikipedia content rarely changes within a year, and re-extracting entities/relations is identical for the same source text. |
| Key | Same URL slug as Tier 1 |
| Content | Both entities list and relations list from LLM extraction. |

```jsonc
{
  "url": "https://en.wikipedia.org/wiki/Robert_E._Lee",
  "extracted_at": 1750001300.2,
  "entities": [ // ~24 entity dicts
    {"entity_type": "person", "node_id": "robert_e_lee", "name": "..."}
  ],
  "relations": [ // ~46 relation dicts
    {"source_id": "robert_e_lee", "target_id": "abraham_lincoln", "relation_type": "..."}
  ]
}
```

**Hit:** Return cached entities + relations — saves two Ollama LLM calls (~150-300s).

**Miss:** Call `extract_facts()` and `extract_relations()`, persist combined result.

### Tier 3 — Neo4j Ingestion Guard

| Property | Value |
|---|---|
| Storage | Query: check Neo4j for nodes with `_source_url = <url>` |
| TTL | N/A (checked live against database) |
| Content | None — pure read-only query |

**Hit:** All entities from this URL already exist in Neo4j. Skip `batch_ingest()` entirely — 0 disk I/O, ~0s.

**Miss:** At least one entity is new or missing. Run full `batch_ingest()`.

## Cache Flow (with all three tiers)

```
get_next_url() → url
    │
    ▼  [T1] page_cache.get(url)
    ┌───────────────────────────────────┐
    │ CACHED (<24h)                    │
    │ → skip WebFetch, text available  │
    └───────────────────────────────────┘
    MISS / stale > 24h
    ▼ fetch_full_page(url)             // Playwright, ~5s + throttle

    ▼  [T2] extraction_cache.get(url)
    ┌───────────────────────────────────┐
    │ CACHED                            │
    │ → skip both LLM calls            │
    │ → entities + relations available  │
    └───────────────────────────────────┘
    MISS
    ▼ extract_facts(text)             // Ollama, ~60-120s
    ▼ extract_relations(entities)     // Ollama, ~90-180s

    ▼  [T3] is_url_ingested(url)?     // Neo4j query
    ┌───────────────────────────────────┐
    │ ALL NODES EXIST                   │
    │ → skip batch_ingest              │
    └───────────────────────────────────┘
    MISS (new/updated entities)
    ▼ batch_ingest(entities, rels)

    mark_completed(url)
```

## Expected Time Savings

| Scenario | Before (no cache) | After (all 3 tiers) |
|---|---|---|
| Re-ingest page completed today | ~250s (fetch + entities + relations + store) | ~0.1s (three cache hits) |
| Page fetched yesterday, entities new | ~210s (entities + relations only) | 0.1s (T1 hit) |
| Fresh page never seen before | ~250s | ~250s (first time, no savings) |

For the current queue of 7 URLs (all already ingested), re-running `batch_ingest.py` would go from **~1800s total to <1s**.

### `wikipedia.py` modification

```python
from hist.cache.page_cache import PageCache

_cache = PageCache()

def fetch_full_page(url, timeout=15):
    cached = _cache.get(url)
    if cached is not None:
        return cached[:6000]

    _throttle()
    text = WebFetch(timeout=timeout).execute(url)
    _cache.put(url, text)
    return text[:6000]
```

### `extractor.py` modification

Two new wrapper functions that check the extraction cache once and return both entities + relations:

```python
from hist.cache.extraction_cache import ExtractionCache

_cache = ExtractionCache()

def extract_with_cache(url, page_text):
    """Check T2 cache first. If miss, call extract_facts + extract_relations."""
    cached = _cache.get(url)
    if cached:
        return cached["entities"], cached["relations"]
    
    entities = extract_facts(page_text)
    relations = extract_relations(entities, page_text)
    _cache.put(url, entities, relations)
    return entities, relations
```

Existing `extract_facts()` and `extract_relations()` remain unchanged for standalone use. `batch_ingest.py` switches to `extract_with_cache()`.
