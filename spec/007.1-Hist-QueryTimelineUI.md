# Phase 7.1 — HIST Query Engine + Timeline UI (v1.0)

## Overview

Extends existing HIST knowledge graph with two back-end components (Cypher query engine, LLM essay formatter) and one web UI panel: a Neo4j Bloom-style timeline visualization plus a question/answer chat interface. Ships as an isolated FastAPI app under `webui/hist`, no Morphos router wiring yet.

---

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Timeline layout | Neo4j Bloom-style horizontal timeline | Events have explicit dates — natural sort axis, user-friendly for history |
| Query approach | Hand-written templates first, LLM fallback second | Deterministic core behavior on common patterns (who/when/where), flexible LLM catch-all for unusual questions |
| Hosting | New FastAPI app under `webui/hist_app.py` | Keeps HIST isolated, no Morphos integration risk |
| Styling | Reuse existing glassmorphism theme (`styles.css`, CSS variables) | Visual consistency, no new design system cost |

---

## Architecture

```
GET /hist                          →  hist_index.html (timeline + Q&A page)
GET /api/hist/graph-data           →  all nodes + edges as JSON for timeline render
GET /api/hist/node-details/<id>    →  full properties + connected edges for side card
POST /api/hist/ask                 →  query engine → Cypher or LLM fallback → formatter → answer
POST /api/hist/ingest/<topic>      →  seeds topic, runs orchestrator.ingest_page()
POST /api/hist/ingest-queue        →  runs all pending queue URLs
GET  /api/hist/stats              →  node/edge counts
```

## Component 1 — Cypher Query Engine

### Hand-written Templates

Each template is a `(pattern, cypher_query)` pair matched against the user's question using keyword triggers. Ordered by priority:

| # | Trigger | Template | Cypher |
|---|---|---|---|
| T1 | `who`, `commander`, `led`, `fought` + event/person name | **Participants** — "Who was at X?" | `MATCH (e:Event {node_id: $id})<-[r]-(p:Person) RETURN p, type(r)` |
| T2 | `when`, `date`, `what year` | **Date lookup** — "When did X happen?" | `MATCH (n) WHERE n.node_id = $id OR n.name CONTAINS $query RETURN n.date, n._ingested_at` |
| T3 | `before`, `after`, `sequence`, `order`, `timeline` | **Chronological events** — sort events by date text | `MATCH (e:Event) WHERE e.date IS NOT NULL RETURN e ORDER BY e.date ASC` |
| T4 | `family`, `parent`, `child`, `relation` + person name | **Family tree** — follow PARENT_OF/CHILD_OF edges up/down N levels | `MATCH path=(p:Person {node_id: $id})-[*1..5]-(r) RETURN path` |
| T5 | `connected`, `links`, `relation` | **Relationships** — all edges touching a node | `MATCH (n {node_id: $id})-[r]-() RETURN n, r` |
| T6 | free text keyword match | **Full-text scan** — search across name/date fields | `MATCH (n) WHERE toLower(n.name) CONTAINS toLower($query) OR toLower(n.summary) CONTAINS toLower($query) RETURN n LIMIT 20` |

Template matching logic:
1. Lowercase the user question
2. Check each trigger list using `any(kw in question for kw in triggers)`
3. Extract entity name via a lightweight LLM call (or regex extraction for capitalized phrases, e.g., "Antietam", "Abraham Lincoln")
4. If template matches, run Cypher, collect nodes + edges
5. Pass raw results to the essay formatter

### LLM Fallback

When no template triggers match:

1. Send question to Ollama `gemma4:12b` with system prompt that instructs generating a single Cypher query
2. Execute generated Cypher in Neo4j
3. If execution fails (syntax / type errors), retry once with error feedback from Neo4j
4. If all Cypher attempts produce zero records, return "No matching data in graph"

Fallback is implemented as `hist/query_engine.py`:

```python
def ask(question):
    """Return (answer_text, evidence_nodes, evidence_edges)."""
    matched = try_templates(question)
    if matched:
        raw = execute_cypher(matched.cypher, matched.params)
    else:
        # LLM fallback
        cypher = generate_cypher(q)
        raw = execute_and_retry(cypher)

    if not raw:
        return "No matching data in the graph."

    answer = format_answer(question, raw)          # LLM formatter pass
    evidence = extract_evidence(raw)                # which nodes/edges were used
    return (answer, evidence["nodes"], evidence["edges"])
```

## Component 2 — Essay Formatter

LLM transform of raw Neo4j rows into a grounded answer. System prompt enforces:

1. Use ONLY the facts provided in `{evidence}` block
2. Cite each sentence with source node using `[node_id]` notation
3. Never infer outside the evidence
4. If question asks for a list/timeline, honor that format
5. Always end with "Sources:" listing Wikipedia URLs from `_source_url` properties

Output: `{answer_text, citations[], sources[]}`

## Component 3 — History Timeline UI

### `webui/hist_index.html`

Single-page layout (no separate dashboard page — timeline IS the dashboard since HIST focuses on temporal data):

**Header region:**
- Search bar for questions (triggers POST `/api/hist/ask`, renders answer below)
- Seed topic input + "Ingest" button (feeds orchestrator pipeline)
- Live stats badge: `8 events · 29 persons · X edges`

**Center — Timeline Canvas:**
- Horizontal timeline axis sorted by year extracted from event `date` property (parses ranges like "1861–1865" into start-year placement)
- Events render as blue pill badges placed on the timeline at their start year; hover shows popup with name, date, source
- Persons render below the timeline in a scrollable "Cast" grid of amber pills — clicking highlights all connected events above
- Edges rendered as subtle connecting lines when both endpoints display on screen
- Zoom/pan controls (mouse wheel zooms along time axis, click-drag pans)
- Filter toggles: [v] Events [v] Persons

**Left side card:**
- Appears when a node is clicked in the timeline
- Shows all properties for that entity (node_id, name, type, date, role, source URL, ingested_at)
- Lists connected nodes via relationship edges with relation type labels
- "Ingest related" button — queues linked Wikipedia pages from entity's `_source_url`

**Right answer panel:**
- Expands when a question is submitted
- Shows formatted answer text
 Below: "Evidence" accordion listing the raw Neo4j nodes and edges that grounded the response (clickable — clicking jumps to node on timeline)
- If no data found, shows suggestion card with topic name + quick-ingest button

### Implementation Details

Timeline rendering uses native Canvas API (no D3 dependency). Layout algorithm:

1. Fetch all events with dates from Neo4j via `GET /api/hist/graph-data`
2. Parse date strings — extract start year (`re.search(r"\b(1\d{3}|20\d{2})\b", date_str)`)
3. Compute viewBox: min_year to max_year, padded 5% each side
4. Place event nodes at x = `(year - min_year) / span * canvas_width`
5. Person nodes placed in grid row below timeline (y offset fixed)
6. Edges drawn only between visible/selected person ↔ event connections

For bloom-style depth-of-field effect: unselected nodes dim to 30% opacity; selected node and direct neighbors stay at full brightness. Hover highlights connected edges with gradient stroke animation using requestAnimationFrame.
