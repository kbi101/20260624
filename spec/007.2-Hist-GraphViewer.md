# Phase 7.2 — HIST Force-Directed Graph Viewer (v0.1)

## Overview

Adds a **force-directed graph visualization** as an alternative view to the horizontal timeline in the HIST UI. The graph view lets users explore the historical knowledge graph from a topological perspective rather than a chronological one.

## Architectural Decisions

| Decision | Choice | Rationale |
|---|---|---|
| View mode | Toggle button switching timeline ↔ graph | Users need both chronological and topological perspectives |
| Layout algorithm | Force-directed (repulsion + spring + centering) | Produces readable layouts that reveal clusters and hubs |
| Physics implementation | Pure JavaScript in browser | No extra dependencies, Canvas-based, ~60fps |
| Node positions | Per-simulation computed, not stored in DB | Graph layout is a view concern, not persisted data |

## Implementation Status (Partial)

### Completed

**Physics simulation in `hist_timeline.js`:**
- `initGraphPositions()` — initializes nodes with random positions, velocities = 0
- `runGraphSimulation()` — per-frame physics:
  - Repulsion: `force = 800 / dist²` (Coulomb-like, with mass scaling)
  - Spring attraction: `force = (dist - 150) * 0.8` toward target length 150px
  - Center gravitation: pulls toward canvas center with `force = 0.1 * dist_to_center`
  - Damping: `velocity *= 0.85` per frame
  - Velocity clamped to max 50px displacement per frame
- `renderGraphView()` — draws edges as faint lines, nodes as colored circles (purple = events, pink = persons) with optional labels

### Missing

1. **UI Toggle Button** — add `[Timeline | Graph]` switch in center panel header
2. **Mouse interaction for graph view:**
   - Click node → select and show detail card (reuse `loadNodeDetail()`)
   - Drag node → apply position offset directly
3. **Simulation stability detection** — stop `requestAnimationFrame` when max velocity < 0.5 for 50 frames
4. **Graph view hover** — highlight connected nodes (depth-1 neighbors)
